---
name: generate-playwright-test
description: Create or update a Playwright test file from a short spec, selected code block, or existing test.
scope: workspace
argument-hint: Provide a short task description or user story, and optionally a target file path.
---

Purpose

- Help generate concise, runnable Playwright `*.spec.js` test files or test snippets that fit the project's conventions.

Inputs (placeholders the prompt will accept)

- `task`: short user story or test intent (e.g., "verify login flow").
- `selected_code` (optional): code or DOM snippet from the editor to base assertions on.
- `target_path` (optional): desired file path to save (e.g., `tests/login.spec.js`).
- `style` (optional): `concise` or `verbose` (default: `concise`).
- `browser` (optional): `chromium|firefox|webkit` (default: project default).

Outputs

- A single file content block with the test code, ready to save as the given `target_path`.
- A suggested filename if `target_path` not provided.
- A 2-line summary describing what the test covers and any required test fixtures or credentials.

Constraints / Conventions

- Use Playwright test runner syntax (import from `@playwright/test`).
- Prefer readable selectors and explicit `await` assertions.
- Keep generated tests idempotent and not reliant on external accounts unless explicitly requested.

Example invocations

- "task: Verify that a user can log in with valid credentials; target_path: tests/login.spec.js"
- "task: Add assertions for the signup form; selected_code: <form id=signup>...</form>"

Ambiguities to clarify after generation

- Should the test use real credentials, mocked data, or a test-only account?
- Preferred placement for fixtures or helper utilities (project-level or test-level)?

Suggested next customizations

- Add switches for `typescript` output or `pageObject` style.
- Add an option to create a test + fixture pair.

When to run me

- Select a user story or code and invoke: "Generate Playwright test".
